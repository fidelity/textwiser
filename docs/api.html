TextWiser: Text Featurization Library

TextWiser is a library that provides a unified framework for text featurization based on a rich set of methods.

_imp

The model pipeline with one embedding and zero or more transformations. The text featurization starts with an embedding, and is sequentially followed by the transformations. Note that the embedding can also be a Compound embedding, which may be a combination of multiple embeddings.

Type

_Sequential

dtype

The requested type of the output. This is useful when working with a downstream task that only uses NumPy, and allows TextWiser to fit into the scikit-learn Pipeline. Defaults to a pytorch tensor.

Type

Union[np.generic, torch.dtype]

fit(x=None, y=None)

Trains the model with the given input data.

Parameters

  • x (Union[None, str, List[str]]) – The training documents to fit the model on.

  • y (Optional) – An optional target argument for possibly doing supervised training. There are currently no supervised models in TextWiser, but the option to do so is here for the future. This also allows TextWiser to fit into the sklearn pipelines.

fit_transform(x, y=None)

Trains the model with the given input data, and transforms the data into embeddings.

This method is equivalent to calling fit first and transform later.

Parameters

  • x (Union[str, List[str]]) – The training documents to fit the model on, and to transform into embeddings.

  • y (Optional) – An optional target argument for possibly doing supervised training. There are currently no supervised models in TextWiser, but the option to do so is here for the future. This also allows TextWiser to fit into the sklearn pipelines.

class BOW(pretrained='default', **kwargs)

Bag of words embeddings.

Computes the frequency of words in each document and uses that as the representation. Supports any attributes that sklearn.feature_extraction.text.CountVectorizer may need.

class Compound(schema: Optional[Union[Dict, list, tuple, str]] = None, pretrained='default', **kwargs)

Compound embeddings derived from a schema.

Compound embeddings cannot be pretrained, you should instead save the whole TextWiser object.

schema

The schema to load the models with. If it is a string and it is a valid path, it is assumed to be the path to a JSON file containing the schema. If it is a string and one of the supported embeddings or transformations, it will be used accordingly. If it is a list or a tuple, it is assumed to be one of the supported embeddings or transformations, and its parameters. If it is a dictionary object, it is assumed to be the JSON schema itself.

Type

The pretrained model to use. Should be None, or be equal to DEFAULT_MODEL.

class Doc2Vec(deterministic: bool = False, tokenizer: Optional[Callable[[str], List[str]]] = None, pretrained='default', **kwargs)

Doc2Vec embeddings.

These embeddings extend the word embeddings to documents using a similar (CBOW) approach. Supports any attributes that gensim.models.doc2vec.Doc2Vec may need.

deterministic

Flag for making the inference procedure deterministic. Due to sampling used in the inference process, multiple calls with the same input results in different vectors. Setting this flag makes the inference deterministic, at the cost of speed. Defaults to false.

Type

bool

tokenizer

A tokenizer function that takes in a document string and returns a list of tokens. Defaults to whitespace splitting.

Type

Callable

pretrained

The name or path of the pretrained model to use. If None, a new model will be trained from scratch when fit is called. Currently, TfIdf, BoW, Doc2Vec, and word2vec models are trainable; everything else will raise an error. Defaults to default, which signals the models to use the default pretrained model, or to train a new model if pretrained model is not available.

Type

class Random(pretrained='default', **kwargs)

Randomized configuration of embeddings.

Creates a random schema, including the possibility of having transformations and concatenations (and even recursive ones). The only restriction is that word vectors need to be pooled immediately within a transformation, whereas they can theoretically be concatenated first and pooled later.

The schemas generated are biased towards sensible values, so as not to create huge models and not waste time with irrelevant configurations. However it is still possible to get large models and even models that do not work, such as a word pooling (which has negative values) followed by NMF (which expects nonnegative values).

class TfIdf(pretrained='default', **kwargs)

TfIdf embeddings.

Computes the frequency of words in each document, balances them with the inverse document frequency, and uses that as the representation. Supports any attributes that sklearn.feature_extraction.text.TfidfVectorizer may need.

class USE(pretrained='default', **kwargs)

Universal Sentence Encoder embeddings.

Uses Google’s Universal Sentence Encoder model, which is in Tensorflow. As such, Tensorflow is a required dependency for using this model. Since the model is pre-built, there are no parameters that can be passed in.

class Word(word_option: textwiser.options.WordOptions = WordOptions.word2vec, pretrained: str = 'default', sparse: bool = False, tokenizer: Optional[Callable[[str], List[str]]] = None, layers: Union[int, List[int]] = - 1, inline_pool_option: Optional[textwiser.options.PoolOptions] = None, **kwargs)

Word embeddings.

These include non-contextual Word2Vec models, such as the original skip-gram model, and the GloVe embeddings, and also contextual embeddings, such as BERT and ELMo. The list of all supported embeddings are in WordOptions, which includes WordOptions.word2vec to support all models available in the Flair framework.

word_option

The word vectorization model to use. Defaults to WordOptions.word2vec.

Type

WordOptions

pretrained

The name or path of the pretrained model to use. If None, a new model will be trained from scratch when fit is called. Currently, only a word2vec model can be trained, everything else will raise an error. Defaults to default, which uses the Flair defaults for each model.

Type

str

sparse

Whether the embeddings layer should be implemented sparsely. Only used with word2vec embeddings. Sparse embeddings are useful for backpropagation, where only a fraction of the embeddings need to be updated. Specifying it results in a significant speedup in the backward call. However, keep in mind that only a limited number of optimizers support sparse gradients: currently it’s optim.SGD (CUDA and CPU), optim.SparseAdam (CUDA and CPU) and optim.Adagrad (CPU).

Type

bool

tokenizer

A tokenizer function that takes in a document string and returns a list of tokens. Defaults to whitespace splitting. Only used with word2vec embedings.

Type

Callable

layers

The hidden layers to use as the word representation in transformers models. Defaults to -1.

Type

Union[int, List[int]]

inline_pool_option

The pooling to be done right after the word embedding for each document (inline). This is the same as having a word embedding immediately followed by a pool transformation.

Specifying a pool transformation here can be significantly less memory intensive if you’re embedding a large dataset at once. If specified, the return value will either be a numpy array or a torch tensor (depending on specified dtype) instead of a list of arrays or tensors. Specifying both inline_pool_option and a following Transformation.Pool for a single Word embedding will result in an error.

Defaults to None, which means no pooling will be done.

Type

Optional[PoolOptions]

class NMF(n_components: int = 10, **kwargs)

NMF Transformation.

Decomposes one nonnegative matrix H into nonnegative matrices U and V such that H = UV^T. Supports any attributes that sklearn.decomposition.NMF may need.

class Pool(pool_option: textwiser.options.PoolOptions = PoolOptions.max, **kwargs)

Pool Transformation.

Pools a list of word embeddings into a single embedding. Should not be specified if inline_pool_option is specified for the corresponding word embedding.

pool_option

The type of pooling operation. Most common is max, which takes the maximum values for all features across the words.

Type

PoolOptions

class SVD(n_components: int = 10, **kwargs)

SVD Transformation.

Decomposes a matrix M into matrices U, S, and V such that M=USV^T. S becomes the matrix of singular values, while U and V matrices are the instance and feature representations using singular values as features. Supports any attributes that sklearn.decomposition.SVD may need.

n_components

The number of factors to find.

Type

int

class UMAP(n_components: int = 10, **kwargs)

Uniform Manifold Approximation & Projection Transformation

Tries to model the manifold with a fuzzy topological structure. Supports any attributes that umap.UMAP may need.

n_components

The number of topics to use.

Type
